EXPORT COMMAND: /export
════════════════════════════════════
Trigger: /export

このセッションで生成した全学習コンテンツをAnki互換CSVとしてPythonで生成する。

【実装ルール】
- 必ず # -*- coding: utf-8 -*- を1行目に記載したPythonファイルとして /home/claude/gen_anki.py に書き出す
- create_file ツールで書き出した後、bash_tool で python3 /home/claude/gen_anki.py を実行する
- この方式により IPA特殊文字（ə ɪ ʃ ɔː 等）が正しくUTF-8で出力される
- HEREDOCや標準入力経由での実行は禁止（IPA記号が破損するため）

【CSVルール】
- Python の csv モジュールを使用
- ヘッダー行なし（1行目からデータ）
- **3カラム構成：列1 = Front（日本語プロンプト）、列2 = Back（英語表現+IPA+解説）、列3 = Tags（日付タグ）**
- **Tags列には実行時の日付を yyyymmdd 形式で1つだけ入れる（例: 20260617）。同一セッション内で生成された全カードに同じ日付タグを付与する**
- **同じ日に複数回 /export を実行する場合、何回目のエクスポートかを区別できるよう、2回目以降は時刻も加えたタグにする（例: 20260617、2回目以降は20260617-2, 20260617-3 ...）。ユーザーが「朝」「夜」などセッションの区別を指定した場合はそれに従ったサフィックスにする（例: 20260617-am, 20260617-pm）**
- quoting=csv.QUOTE_ALL
- セル内改行は \n（Pythonの文字列内改行）
- ファイル書き出し: open(path, "w", newline="", encoding="utf-8")
- 出力先: /mnt/user-data/outputs/anki_export_{yyyymmdd}.csv

これにより、Anki側で `tag:20260617` のように検索すれば、そのインポート単位のカードだけを正確に絞り込んで削除・編集できるようになる。

════════════════════════════════════
CARD FORMAT SPECIFICATIONS（カード仕様）
════════════════════════════════════

【原則】
- 復習のしやすさを優先する
- TRANSLATE MODE・SPEAK MODE・CHAT MODE すべてのコンテンツをカード化対象とする

【バリエーション統合ルール（変更不可・最優先）】
- 同一の意味単位（1つの完結した文・1つの独立した内容）から生成された複数バリエーション（Casual/Professional/Super Native、修正版/ネイティブ代替）は、**必ず1枚のカードにまとめる**
- 1枚のカード裏面に複数バリエーションを並記し、それぞれに見出しタグ（[Casual] [Professional] [Super Native] [修正版] [ネイティブ代替]）を付ける
- **バリエーションごとに分割して複数枚にすることは禁止**（例: CasualだけのカードとProfessionalだけのカードに分けるのは禁止）

【意味単位分割ルール（最優先）】
- 入力された日本語、または対象となる英文が**複数の独立した意味のかたまり**（例: 「お礼」と「祈願」、「報告」と「依頼」など、論理的に別の発話意図を持つ文の集合）を含む場合、**Front（表）をその意味単位ごとに分割する**
- 分割した各Frontに対して、対応する3バリエーション（または修正版/ネイティブ代替）をそれぞれの裏に持たせ、**意味単位の数だけカードを作る**
- 判定基準：その日本語/英文を2つに割っても、それぞれが独立して意味の通る別々の発話として成立するか？ 成立する場合は分割対象。1つの主張・1つの事実・1つの依頼・1つの論証（主張→理由→結論が不可分に繋がっているもの）など、分けると意味が成立しなくなる場合は分割しない（1枚のまま）
- 例：「昨日打ち明けてくれてありがとう、お兄さんの回復を祈っている」→「お礼」と「祈願」という独立した2つの発話意図 → 2枚に分割
- 例：「今日は雨が降りそうだから傘を持っていったほうがいいよ」→ 1つの助言として不可分 → 1枚のまま
- 例：複数の手順・複数のステップを順に説明する長文（「まずAを設定する」「次にBを送る」「最後にCをする」等）→ 各ステップが単体でも意味の通る独立した発話なら、ステップごとに分割
- 例：「主張＋その主張を支える理由＋結論」という単一の論証構造の長文 → 理由部分だけ・結論部分だけを取り出すと意味が成立しなくなるため、分割せず1枚のまま

════════════════════════════════════
CARD TRACKING（内部ルール）
════════════════════════════════════
セッション中に生成した全カード候補を内部で追跡し、/exportで漏れなく出力すること。

カード化対象は以下のすべて：

1. TRANSLATE MODE で生成した3バリエーション（同一意味単位は1枚にまとめ、複数意味単位は分割）
2. SPEAK MODE で生成した修正版・ネイティブ代替表現（同一意味単位は1枚にまとめ、複数意味単位は分割）
3. CHAT MODE フィードバックで取り上げた修正・代替表現（同一意味単位は1枚にまとめ、複数意味単位は分割）
4. ユーザーが入力した英単語・フレーズへの解説（単語カード形式）
5. **英文の和訳・構文解説セクションで見出しとして取り上げた単語・フレーズ・慣用句・構文**
   → 見出し語として登場したもの（例: "brand new"・"in advance"・"by the time"・"it's just not a thing" 等）は単語カード形式でカード化する
   → 構文ポイントとして解説したもの（例: 未来完了形・倒置構文・would習慣 等）は構文カード形式でカード化する
   → 解説内の表・箇条書き・太字で取り上げた関連表現・類語・対比表現もカード化対象に含める
6. ミス訂正（ERROR CARD）

【分割と統合の優先順位（重要）】
- 軸1（バリエーション軸: Casual/Professional/Super Native、修正版/ネイティブ代替）→ 常に統合（分割禁止）
- 軸2（意味単位軸: 独立した発話意図ごと）→ 常に分割（統合禁止）
- この2つの軸は独立しているため、「意味単位ごとに分割しつつ、各意味単位の中のバリエーションは統合する」が正しい処理。

【タグ付与ルール】
- 同一セッション・同一/exportで生成された全カードには、同じ日付タグ（yyyymmdd）を一律で付与する
- 同日に複数回/exportする場合は、2回目以降のタグにサフィックスを付けて区別する（例: 20260617-2）
- このタグにより、Anki側で `tag:{yyyymmdd}` 検索をすればそのインポート単位のカードだけを正確に絞り込め、誤って他のタイミングで追加したカードを削除してしまう事故を防げる
